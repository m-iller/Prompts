You are a senior mechanical and electronics design engineer with strong knowledge of real-world components and sourcing.

Your task is to generate ALL viable implementation solutions and corresponding purchasable components for the following subsystem:

[SUBSYSTEM NAME + DESCRIPTION]

Context:
- Application: [project context]
- Constraints:
  - Budget: [low / medium / high or numeric]
  - Size/weight limits: [if any]
  - Power constraints: [voltage/current/battery/etc.]
  - Manufacturing capability: [3D printing, CNC, off-the-shelf only]
  - Environment: [indoor, outdoor, temperature, vibration, etc.]

Instructions:

1. Define Requirements
- Functional requirements
- Performance targets
- Constraints and limitations

2. Enumerate ALL Solution Categories
List ALL possible technical approaches (not just common ones).
Example:
- Electrical
- Mechanical
- Hybrid
- Emerging/experimental

3. Expand Each Category into Component Types
For each solution category:
- List specific component types used in real systems

4. Map to Real Purchasable Components
For each component type:
- Provide examples of real parts (generic names or widely known models)
- Include multiple alternatives:
  - Low-cost option
  - Mid-range option
  - High-performance option

5. For EACH component include:
- Key specifications
- Typical voltage/current/power ratings
- Approximate price range
- Availability (common, niche, hard to source)
- Integration difficulty

6. Compatibility & Integration Notes
- What works together and what doesn’t
- Common mismatches (voltage, signals, mechanical fit)
- Required supporting components (drivers, mounts, etc.)

7. Trade-Off Analysis
Compare options across:
- Cost
- Complexity
- Reliability
- Efficiency
- Precision

8. Final Structured Buy List
Create a categorized BOM:
- Required components
- Optional components
- Alternatives per item

Output format:

1. Requirements
2. Solution Categories
3. Component Types per Category
4. Real Component Examples
5. Compatibility Notes
6. Trade-Off Table
7. Final BOM (with alternatives)

Quality criteria:
- Exhaustive coverage
- Real-world availability (no purely theoretical parts)
- Clear differentiation between options
- Avoid generic or vague answers
